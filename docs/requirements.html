Requirements

Authorization & authentification

Sign In:

  • proceed to main page in case of success
  • show error in case of failure

Sign Up:

  • check uniqueness of login
  • compare password and repeated password
  • proceed to main page in case of success
  • show error in case of failure

Search Page (Main page)

  • show filter bar at the top (locality from and locality to - 2 text fileds, 2 date pickers for date range)
  • show search button
  • show results in table view with customized cells, containing info about tickets
  • provide possibility to book ticket

Map page

  • show map
  • show text field for departure point
  • show find button
  • show error if no locality with entered name exists
  • if entered locality exists - show it as D - departure on the map and available localities to as A - arrival

Booked Ticket Page

  • show booked tickets as table view
  • in customized cell show all info about ticket
  • make it possible to unbook ticket if it is still actual

Navigation

  • provide an app with a menu bar (Search Page, Map Page, Booked Ticket Page)

User stories

Authorization & authentification

As a user, I want to sign up, so that I can use your app further

As a user, I want to sign in, so that I can use your app

Search Page (Main page)

As a user, I want to view tickets, so that I can choose appropriate one

As a user, I want to filter tickets, so that I can easier find appropriate one

As a user, I want to book chosen ticket, so that I can buy it later

Map page

As a user, I want to see available directions from chosen locality, so that I can know, where I can get

Booked Ticket Page

As a user, I want to see booked tickets, so that I can make sure I booked them

As a user, I want to unbook tickets I do not need (if it is available), so that other people can use them

Navigation

As a user, I want to navigate through pages, so that I can get all information
